About the Weather API Project

This project is a web-based weather application that displays the current weather conditions for a predefined list of major cities around the world.

Key Features:

  • Global Weather Dashboard: Displays weather cards for multiple cities simultaneously.
  • Real-time Data: Fetches up-to-date weather information from the Open-Meteo API.
  • Visually Appealing UI: The user interface is styled with custom CSS and Bootstrap, providing a clean and modern look.

Technologies Used:

Backend:

  • Node.js: A JavaScript runtime environment for building the server-side application.
  • Express.js: A web application framework for Node.js, used to create the API endpoints.
  • Axios: A promise-based HTTP client for making requests to the Open-Meteo API.
  • dotenv: A module to load environment variables from a .env file.
  • cors: A Node.js package for providing a Connect/Express middleware that can be used to enable CORS with various options.

Frontend:

  • HTML, CSS, JavaScript: The core technologies for building the user interface.
  • Bootstrap: A popular CSS framework for creating responsive and mobile-first websites.
  • Axios: Used on the client-side to fetch data from the backend API.

Architecture:

The application follows a simple client-server architecture:

  1. Frontend: The public directory contains the static frontend files (index.html, style.css, script.js). The JavaScript code makes an API call to the backend to fetch the weather data.
  2. Backend: The server.js file sets up an Express server with a single API endpoint (/api/weather). This endpoint fetches weather data from the Open-Meteo API for a list of hardcoded cities and returns it to the frontend.
